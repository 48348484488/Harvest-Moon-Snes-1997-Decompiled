# Pseudocode - Event Script Content Group Lookup

```c
const uint24 EventScript_MasterGroupPointerTable[0x48] = {
    // EventScriptGroup_00 .. EventScriptGroup_47
};

uint24 EventScript_GetGroupTable(uint8 group_id) {
    // master table is at $B3:8000
    return EventScript_MasterGroupPointerTable[group_id];
}

uint24 EventScript_GetEntry(uint8 group_id, uint8 entry_id) {
    uint24 group_table = EventScript_GetGroupTable(group_id);
    uint8 bank = bank_of(group_table);

    // each group table stores local 16-bit offsets inside its own bank
    uint16 local = read16(group_table + entry_id * 2);
    return make24(bank, local);
}
```

## Notes

- Pass 55 names the master table as `EventScript_MasterGroupPointerTable`.
- Group labels follow `EventScriptGroup_XX`, where `XX` is the group id used by the engine/content tables.
- Most groups have 16 entries. `EventScriptGroup_00` is a larger 136-entry table, and `EventScriptGroup_47` has 32 entries.
- A group entry can repeat another target. This is normal and usually means that several variants/directions share the same subscript.
- The catalog generated by `tools/event_script_content_catalog.py` should be regenerated whenever the content banks are repointed or edited.
